Mathematical reasoning, made inspectable

Find the exact step where a solution state drifts.

StateScope compares model-carried text state with a typed, operation-derived ledger. Every transition is checked against a SymPy oracle, then retained for inspection, intervention, and replay.

1. SelectChoose a problem, model, execution route, and comparison mode.

3. LocalizeCAS verification marks the first component-level divergence.
4. InterveneEdit retained steps, branch repeatedly, and measure recovery.

More than final-answer scoring

StateScope turns a solution into an auditable sequence of state updates and separates operation validity from failures in carrying state forward.

Controlled state ownership

The same verified operation schedule isolates who owns the solution state.

Step-level CAS verification

SymPy identifies the first mismatched component without an LLM judge.

Repeatable intervention

Correct or perturb any retained step, continue, revisit, and intervene again.

Recovery analysis

Measure whether targeted repair restores faithful reasoning and the answer.

One schedule. Two state owners.

The comparison keeps operations fixed so the experimental variable is explicit.

SYSTEM C

Model-carried state

The model owns textual state, which can omit, overwrite, or carry stale information.

SYSTEM D

Ledger-derived state

A typed ledger re-derives state from verified operations and refuses direct state edits.
