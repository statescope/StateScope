StateScope

Detecting, localizing, and explaining solution-state drift in mathematical reasoning. Every step is verified against a SymPy oracle.
StateScope turns a mathematical solution into an inspectable sequence of verified state updates.
1Choose a run Pick a problem, model, and execution route. Every model uses the same typed operation protocol, and every step is checked against the CAS oracle.
2Compare state ownership System C carries the model's claimed text state and can drift. System D re-derives state in a typed ledger. Controlled comparison keeps the operation schedule fixed so this ownership difference is isolated.
3Watch the reasoning unfold Use Step through for one operation at a time, or Run both for the complete trace. Autonomous diagnostic mode lets the model choose its own operation schedule.
4Intervene and explore Use what-if at any step to edit an operation, or edit System C's claimed state directly. Continue with the model or re-derive on the ledger. Failed attempts and every branch stay editable. Use ↻ fresh for new parameters.
